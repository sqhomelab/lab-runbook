# lab-runbook

Narrative record of incidents and procedures for the homelab/auxlab infra.
This is NOT the infrastructure-as-code source of truth — that lives in the Ansible repo.

## Structure
- `incidents/` — something broke; what happened, why, how it was fixed
- `procedures/` — routine/repeatable maintenance tasks, not failure-driven
- `reference/` — architecture snapshots, decisions (optional)

## Naming
- Incidents: `incidents/YYYY-MM-short-slug.md`
- Procedures: `procedures/verb-noun.md`

Copy TEMPLATE.md for every new entry.
